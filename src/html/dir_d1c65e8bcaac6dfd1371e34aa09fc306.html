file  task_planner.cpp
 The master of all the project, and the entry point of the workflow. It invokes services from all the other nodes in the following order: 1) vision_node detects the bounding boxes. 2) pointcloud_node uses such information to estimate the blocks' poses. 3) desired_poses_node provides the target pose for each block. 4) motion_planner recieves detected and desired pose for each block and moves it.
